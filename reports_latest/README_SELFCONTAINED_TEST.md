# PMU Self-Contained 5-Cycle Test

## Overview

This test validates the PMU system with 1500 hardcoded samples (5 power cycles at 50 Hz, 300 samples/cycle) and displays the output as a hex dump of 76 bytes × 5 packets.

## Files

### Core Test Files (NEW)
1. **generate_test_constants.py** - Python script to generate VHDL constants package
2. **test_data_constants_pkg.vhd** - Auto-generated VHDL package with 1500 packets (generated by script)
3. **tb_pmu_selfcontained_5cycles.vhd** - Self-contained testbench
4. **compile_selfcontained_test.sh** - Compilation script
5. **run_complete_test.sh** - Full automation script (generate + compile + simulate)
6. **run_selfcontained_test.tcl** - TCL script for Vivado GUI

### Dependencies (Existing)
- **8.vhd** - Top-level DUT (pmu_system_complete_256)
- **medhavi.csv** - Source data (first 1500 rows used)
- All supporting PMU modules (resampler, DFT, CORDIC, etc.)

## Test Configuration

- **Total samples:** 1500 (from medhavi.csv)
- **Power cycles:** 5 (50 Hz, 300 samples/cycle)
- **Sample rate:** 15 kHz (66.67 µs between samples)
- **Clock frequency:** 100 MHz
- **Input packet format:** 128-bit AXI stream
  - [127:120] = 0xAA (SYNC)
  - [119:104] = Ch0 (16-bit, signed → unsigned hex)
  - [103:88] = Ch1
  - [87:72] = Ch2
  - [71:56] = Ch3
  - [55:40] = Ch4
  - [39:24] = Ch5
  - [23:16] = 0x55 (CHECKSUM)
  - [15:0] = 0x0000 (Reserved)
- **Output packet format:** 76 bytes (19 words × 4 bytes)
- **Expected packets:** 5

## Verification Criteria

**Pass Conditions:**
- ✅ All 1500 samples injected successfully
- ✅ Exactly 5 packets captured
- ✅ Each packet is 76 bytes (19 words × 4 bytes)
- ✅ SYNC = 0xAA01 in all packets
- ✅ FRAMESIZE = 0x004C (76 bytes) in all packets
- ✅ All 6 channels have non-zero magnitude values
- ✅ STAT field = 0xC010 (normal operation)
- ✅ No simulation errors or crashes

**Expected Values (approximate):**
- CH1-3 Magnitude: ~0x0011A3XX (voltage channels)
- CH4-6 Magnitude: ~0x0009E6XX (current channels)
- Frequency: Converges to ~0x00320000 (50 Hz in Q16.16 format)
- CRC: Non-zero values in packets 2-5 (packet 1 may be 0x0000)

## Simulation Time

- **Simulation time:** ~100 ms (1500 samples × 66.67 µs)
- **Real time:** 2-3 minutes (depends on system performance)

## Troubleshooting

### Error: "CSV file not found"
Ensure `medhavi.csv` exists at:
```
/home/arunupscee/Desktop/xtortion/c37 compliance/medhavi.csv
```

### Error: "Compilation failed"
Check that all dependent VHDL files are present:
- circular_buffer.vhd
- sample_counter.vhd
- cycle_tracker.vhd
- ... (see compile_selfcontained_test.sh for full list)

### Error: "Entity not found"
Make sure `8.vhd` contains the entity `pmu_system_complete_256`

### No packets captured
- Check that the DUT is correctly instantiated
- Verify reset signal is properly released
- Check AXI handshake signals (tvalid/tready)

### Simulation hangs
- The test should complete in ~120 ms simulation time
- If it hangs, check for deadlocks in AXI handshake
- Use Ctrl+C to abort and check the log

## Data Flow

1. **Python Script** reads medhavi.csv (1500 rows)
2. **Python Script** converts 6 signed 16-bit channels → 128-bit packets
3. **Python Script** generates VHDL package with constant array
4. **Testbench** reads packets from constant array
5. **Testbench** injects packets via AXI stream (15 kHz rate)
6. **DUT** processes packets (resample, DFT, CORDIC, format)
7. **Testbench** captures 19-word output packets
8. **Testbench** displays hex dump of all 5 packets

## Notes

- No file I/O during simulation (all data hardcoded)

- Self-contained (no external dependencies during simulation)
- Console output only (no CSV file generation)
- Designed for IEEE C37.118 compliance testing
